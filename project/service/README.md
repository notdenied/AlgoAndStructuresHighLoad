# ⚡ Инструкция по развертыванию сервиса Smart Grid

Этот сервис объединяет LSM-Tree, Append-only Log и Inverted Index в распределенную систему мониторинга электросети.

## 📋 Предварительные требования
- Docker
- Docker Compose

## 🚀 Быстрый запуск

1. Перейдите в директорию сервиса:
   ```bash
   cd project/service
   ```

2. Соберите и запустите контейнеры:
   ```bash
   docker compose up -d --build
   ```

3. Откройте панель управления в браузере:
   **[http://localhost:8000](http://localhost:8000)**

## 🏗️ Компоненты системы

- **Collector (Центральный узел)**: Принимает телеметрию, сохраняет её в LSM-хранилище, управляет очередью команд и индексирует отчеты об инцидентах.
- **Sensors (Датчики)**: Три независимых эмулятора датчиков (`sensor_1`, `sensor_2`, `sensor_3`), которые:
  - Шлют данные вольтажа и тока каждые 2 секунды.
  - Динамически отправляют отчеты об ошибках (критическое падение напряжения и т.д.).
  - Опрашивают команды и переключают режим (ВКЛ/ВЫКЛ).
- **Admin Dashboard**: Веб-интерфейс для мониторинга статуса датчиков, отправки команд и поиска по логам.

## 🛠️ Полезные команды

- **Просмотр логов в реальном времени**:
  ```bash
  docker compose logs -f
  ```

- **Масштабирование датчиков**:
  Чтобы добавить больше датчиков, отредактируйте `docker-compose.yml` и добавьте новые секции для сенсоров с уникальными `SENSOR_ID`.

- **Проверка сохраненных данных**:
  Файлы LSM-дерева и логи сохраняются в Docker-волюме `service_collector_data`. Чтобы увидеть их на хосте:
  ```bash
  ls -R project/service/data/lsm_storage
  ```

## 🔍 Проверка функций
- **Live Status**: В админке вы увидите таблицу датчиков. Если отправить команду `OFF`, статус датчика окрасится в красный, а логи телеметрии от него прекратятся.
- **Полнотекстовый поиск**: Введите в поисковую строку слово "напряжение" или "перегрев", чтобы найти ошибки, которые динамически прислали датчики.
