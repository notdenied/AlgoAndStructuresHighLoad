# Smart Grid: Система обработки телеметрии умных электросетей

Данный проект представляет собой высоконагруженную IoT-систему для сбора и анализа данных с 500 000 умных счетчиков. Архитектура разработана с учетом отказоустойчивости (AP-дизайн) и высокой интенсивности записи.

## Основные компоненты

1.  **LSM-Tree Storage** (`task1`):
    *   Специализированное хранилище для данных телеметрии.
    *   Реализованы: WAL (Write-Ahead Log), фильтры Блума, разреженные индексы и компакция.
2.  **Incident Index & Command Bus** (`task2`):
    *   **Inverted Index**: Позволяет диспетчерам мгновенно находить отчеты ремонтных бригад по ключевым словам.
    *   **Append-only Log**: Лог команд для управления фидерами и подстанциями с поддержкой смещений (offsets).
3.  **MapReduce & Spark Billing** (`task3`):
    *   Распределенная обработка 5-10 ГБ данных телеметрии для расчета счетов (биллинг).
    *   Поддержка как локального MapReduce (multiprocessing), так и кластера Apache Spark в Docker.

## Развертывание кластера Spark

Для запуска Spark-кластера используйте Docker Compose:

```bash
cd project/task3
docker compose up -d
```

Spark Master будет доступен по адресу [http://localhost:8080](http://localhost:8080).

## Тестирование

Для автоматической проверки всех компонентов системы выполните скрипт:

```bash
bash project/run_all_tests.sh
```

### Структура тестов (дир `project/tests`):
*   `test_lsm.py`: Проверка долговечности (WAL), поиска и компакции.
*   `test_incident_index.py`: Проверка полнотекстового поиска по отчетам.
*   `test_command_bus.py`: Проверка записи и чтения команд с разных смещений.
*   `test_map_reduce.py`: Интеграционный тест обработки телеметрии.

## Генерация данных

Для создания тестового набора данных телеметрии:

```bash
python3 project/task1/generator.py
```
